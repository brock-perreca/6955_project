# `assets/` — static project assets

| Subdirectory | Contents |
|---|---|
| [`mjcf/`](mjcf/) | MuJoCo MJCF (XML) model files. The active `--scale_model` MJCF lives here. |
| [`reference/`](reference/) | Reference motion data consumed by the active pipeline. |

Active code resolves these paths via
`PROJECT_ROOT = Path(__file__).resolve().parents[2]` from
`src/walker2d/*.py` (i.e. it points to the repo root, then `assets/...`).

## `mjcf/`

| File | Status |
|---|---|
| `walker2d_subject1.xml` | Subject-1-scaled Walker2d. **Currently missing on this checkout** — required for `--scale_model` and the default `--xml` for `render_phase.py`. Regenerate or copy from the user's other machine. |
| `walker2d_custom.xml` | Legacy custom MJCF. Predates the decision to stick with stock Walker2d-v4. Don't reference from active code. |

The stock `walker2d.xml` is **not** stored here — it's pulled from
gymnasium's bundled assets at runtime when `xml_file="walker2d.xml"`
is passed.

## `reference/`

| File | Generated by | Format |
|---|---|---|
| `gait_cycle_reference.npy` | `src/walker2d/extract_gait_cycle.py` | `(N, 6)` float32, radians, Walker2d sign convention, **50 Hz**. Resampled to 125 Hz at env-init time inside `Walker2dPhaseAware.__init__`. |

Joint order: `[hip_r, knee_r, ankle_r, hip_l, knee_l, ankle_l]`.

See [`../docs/DATA_SOURCES.md`](../docs/DATA_SOURCES.md) for details on
the Ulrich source data and how the cycle is extracted.
